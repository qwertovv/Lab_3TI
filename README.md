# Loop Invariant Checker

## Описание проекта

Loop Invariant Checker - это интерактивное WPF-приложение для проверки корректности циклических алгоритмов с использованием методов формальной верификации. Проект разработан в рамках лабораторной работы по изучению инвариантов циклов, вариант-функций и тотальной корректности программ. Приложение позволяет наглядно демонстрировать выполнение трех различных циклических алгоритмов с пошаговой проверкой инвариантов и отслеживанием вариант-функций, обеспечивая визуализацию процесса верификации циклов в реальном времени.

Основные возможности приложения включают: работу в трех режимах (вычисление суммы префикса массива, подсчет элементов превышающих порог, нахождение максимального элемента), генерацию тестовых массивов с настраиваемыми параметрами, пошаговое выполнение циклов с подсветкой изменяемых переменных, автоматическую проверку инвариантов до и после каждого шага, визуализацию убывания вариант-функций, а также подробное логирование всех операций. Приложение предназначено для образовательных целей и может использоваться для изучения основ формальной верификации программного обеспечения.

## Архитектура проекта

Проект реализован на платформе .NET 8 с использованием WPF (Windows Presentation Foundation) и следует принципам MVVM (Model-View-ViewModel). Решение состоит из двух основных проектов: основного приложения и модуля тестирования. В основном проекте выделяются следующие ключевые компоненты: Models содержат классы данных для хранения информации о массиве (ArrayData), состоянии цикла (LoopState) и инвариантах (InvariantData). ViewModels реализуют бизнес-логику приложения в классе MainViewModel, который управляет выполнением циклов, проверкой инвариантов и обновлением состояния интерфейса. Views представлены основным окном приложения (MainWindow) и окном справки (HelpWindow), обеспечивающими пользовательский интерфейс.

Логическая архитектура построена вокруг центрального механизма выполнения циклов, который включает инициализацию параметров, пошаговое выполнение операций, проверку инвариантов через специализированные методы для каждого режима работы, и отслеживание вариант-функций. Система тестирования реализована с использованием xUnit framework и покрывает основные сценарии работы приложения, включая проверку постусловий, монотонного убывания вариант-функций и сохранения инвариантов на протяжении всего выполнения циклов. Приложение использует привязку данных WPF для синхронизации состояния между ViewModel и пользовательским интерфейсом, а также механизм команд для обработки пользовательского ввода.

## Системные требования

### Минимальные требования для запуска:
- **Операционная система**: Windows 10 версии 1809 или новее, Windows 11
- **.NET Runtime**: .NET 8.0 Desktop Runtime или выше
- **Процессор**: x64-совместимый, 1 ГГц или выше
- **Оперативная память**: 2 ГБ
- **Место на диске**: 50 МБ свободного пространства

### Рекомендуемые требования:
- **Операционная система**: Windows 10 версии 21H2 или новее, Windows 11
- **SDK**: .NET 8.0 SDK для сборки из исходного кода
- **Среда разработки**: Visual Studio 2022 версии 17.8 или новее
- **Процессор**: x64-совместимый, 2 ГГц или выше
- **Оперативная память**: 4 ГБ
- **Место на диске**: 100 МБ свободного пространства

### Зависимости проекта:
- **CommunityToolkit.Mvvm** (8.2.1) - для реализации MVVM паттерна
- **Microsoft.NET.Test.Sdk** - для модульного тестирования
- **xUnit** (2.6.6) - фреймворк для тестирования
- **xunit.runner.visualstudio** - интеграция тестов с Visual Studio

### Поддерживаемые конфигурации сборки:
- **Целевая платформа**: .NET 8.0-windows
- **Архитектура**: x64
- **Режим сборки**: Debug/Release
